Custom Tile Server Configuration

The Custom Tile Maps provides functionality to add map that uses web tiling scheme.

Here is the explanation of how to add, configure and remove Custom Tile map in Hale studio. To work with Custom Tile maps, first open Map perspective using Open Perspective tool menu reside in the right corner of the studio.

Custom tile map is a default map in HALE. By default you will see, Stamen Terrain map [Map tiles by Stamen Design, under CC BY 3.0. Data by OpenStreetMap, under CC BY SA.]

To add the Custom Tile maps in to Map perspective, click on the View Menu button and then heads to Custom Tile Maps→Add.

By clicking on Add button, a dialog will be appeared, which asks for the Name and URL Pattern. You have to insert a unique name and URL pattern from where tiles will be fetched.

Please note, here name must be unique from any other Custom Tile map configurations and URL pattern must include {z}, {x} and {y} literals which will be replaced by the HALE as Zoom level, x-coordinate and y-coordinate respectively while fetching the tiles. To go to the next configuration page, click on Next button in the dialog or click Finish directly to skip next configuration pages.

On this page, You have to define the maximum zoom level for the map (default is 16) and attribution text of the map server. Click Finish to complete the configuration. Please note, before using a map you should make sure that you doing so is in compliance with the respective terms of use.

After finishing configuration, Custom Tile maps will be loaded to Map perspective and also added in to the view menu.

You can again reconfigure the map added by you, by clicking on the name you configured before. It will reopen the same dialogs as per above in same order. You can change the configurations in those dialogs.
